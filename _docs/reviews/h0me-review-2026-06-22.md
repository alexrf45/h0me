## Executive Summary

Headline: a **Talos provider `0.12.0-alpha.4` migration** of `_infra/modules/talos-pve`
is staged in the working tree (uncommitted, unapplied). It swaps the
`talos_machine_configuration_apply` + `talos_machine_bootstrap` + `time_sleep`

1Password item is absent, so the cluster could not be queried.

Recommended next sprint: **land the Talos migration** (manual SOPS tfvars edits →
`terraform apply` via 1Password) and confirm the cluster comes up clean, then
write the Kyverno worker-label policy that the deleted `worker-labels.tf` left open.
