## Executive Summary

Headline: **the platform is green end-to-end.** Every Flux kustomization is Ready,
**every HelmRelease is Ready** (incl. `kube-prometheus-stack`, which was still
installing on 06-23), all PVCs are **Bound**, and there are **zero** pods outside
Running/Completed. The entire 06-23 MEDIUM/observability tier (N1 dynamic PVCs, N2
stale pods, N3 wildcard cert, O1 observability, M2 k8s-1.36, N5 worker label, N4
hubble) is **resolved**. The wildcard cert is Ready on the **production** Let's
Encrypt issuer.

The new work since 06-23 was twofold: (1) the storage fallout was chased to root cause
across four post-mortems (freshrss zvol/volumeHandle mismatch, iSCSI initiator-group
mismatch, cf-token 1Password property mismatch, freshrss missing data dirs) and fixed;
(2) **GW1** — Cilium 1.20's Gateway controller refused to start because the Gateway API
CRDs were the wrong channel (standard v1.4.0); vendored v1.5.1 under
`_global/crds/gateway-api/` and the GatewayClass/Gateway are now Accepted/Programmed.

What's left is hygiene + resilience + design, not breakage: **M1** (PDBs still only on
freshrss — now a prerequisite for A1), **HY1** (delete the deprecated `talos-pve-v3.1.0`
module, still present), **HY2** (stale doc paths), **N6** (Cilium pinned to a
pre-release), and two carried design tasks — **CI1** (Tailscale CI/CD) and **A1**
(Proxmox upgrade automation).

Recommended next sprint: **M1 PDB coverage** (unblocks A1), then hygiene (HY1/HY2) and
the N6 Cilium-pin decision.
